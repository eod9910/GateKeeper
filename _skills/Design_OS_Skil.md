# DESIGN OS SKILL
# Visual + System + Accessibility Operating System
# Non-Negotiable UI Constitution

---

# 0. DESIGN PHILOSOPHY

Design is clarity under constraint.

Every interface must be:
- Clear
- Intentional
- Structured
- Accessible
- Consistent
- Calm

If it adds noise, remove it.
If it adds friction, simplify it.
If it adds ambiguity, clarify it.

---

# 1. STRUCTURE FIRST (Before Visuals)

Before generating UI, define:

- Page purpose
- Primary user action
- Secondary actions
- Content hierarchy
- Information density level (low / medium / high)

Do not design before structure is defined.

---

# 2. SPACING SYSTEM (MANDATORY SCALE)

Only use this spacing scale:

4
8
12
16
24
32
48
64
96

Never invent random spacing.

Whitespace is a design tool.
Use it intentionally.

---

# 3. LAYOUT RULES

- Max content width: 1120px desktop
- Content padding: 24px desktop / 16px mobile
- Use vertical rhythm
- No edge crowding
- No uneven alignment
- No inconsistent padding

Define alignment per page:
- Left-aligned content for productivity apps
- Centered layout for landing pages

---

# 4. TYPOGRAPHY SYSTEM

Define:

- H1 (36–44px)
- H2 (28–32px)
- H3 (20–24px)
- Body (16–18px)
- Caption (12–14px)

Rules:

- Clear hierarchy
- No arbitrary font sizes
- Line-height 1.5–1.7 for body
- Never reduce readability for aesthetics
- Use semantic structure (H1, H2, form labels, etc.)

---

# 5. COLOR SYSTEM

Must define:

- Primary color
- Secondary color
- Accent color
- Background color
- Surface color
- Text primary
- Text muted
- Border color
- Success / Warning / Error states

Rules:

- Primary = main action
- Secondary = supporting UI
- Accent = rare emphasis
- Do not rely on color alone for meaning
- Maintain strong contrast ratios
- Avoid color overload

Maximum 2–3 dominant colors per screen.

---

# 6. COMPONENT SYSTEM (MANDATORY)

All UI must be component-based.

Components must define:

- Default state
- Hover state
- Active state
- Focus state
- Disabled state
- Loading state
- Error state (if applicable)

Required base components:

- Button (primary / secondary / ghost)
- Input
- Select
- Checkbox
- Radio
- Card
- Modal
- Toast / Notification
- Table (if data-heavy)
- Empty state block

No one-off styling allowed.

---

# 7. EMPTY / LOADING / ERROR RULES

Every screen must define:

- Empty state
- Loading state
- Error state

Empty state must include:
- Explanation
- Next step CTA
- No dead ends

Loading state must:
- Communicate progress
- Avoid layout shift

Error state must:
- Clearly explain issue
- Provide resolution path
- Never blame the user

---

# 8. ACCESSIBILITY NON-NEGOTIABLE

UI must be:

Perceivable  
Operable  
Understandable  
Robust  

Checklist:

- Alt text for images
- No text baked into images
- Proper semantic HTML
- Keyboard navigation works
- Focus indicators visible
- Minimum touch targets 44px
- Contrast ratio compliant
- Reduced motion support
- Avoid motion overload
- Screen reader friendly labels

Accessibility improvements benefit all users.

---

# 9. INTERACTION DISCIPLINE

- Transitions 150–250ms
- Avoid excessive animation
- Microinteractions must clarify state change
- No decorative motion without purpose
- Feedback must be immediate

User must always know:
- What happened
- What changed
- What to do next

---

# 10. INFORMATION DENSITY CONTROL

For productivity apps:

- Reduce decoration
- Emphasize clarity
- Avoid unnecessary icons
- Avoid redundant labels
- Avoid clutter

For marketing pages:

- Allow larger spacing
- Allow visual hierarchy emphasis
- Still maintain clarity

---

# 11. BRAND ALIGNMENT

Before designing:

- Identify brand tone (minimal / bold / playful / premium)
- Define visual direction
- Define color personality

Never mix styles across screens.

Brand consistency > creative improvisation.

---

# 12. DESIGN QA OUTPUT REQUIREMENT

Before finalizing UI, AI must output:

1. Spacing scale validation
2. Typography validation
3. Color role validation
4. Component state validation
5. Accessibility checklist confirmation
6. Empty/loading/error state confirmation

If any are missing, revise.

---

# 13. NO DESIGN WITHOUT INTENT

Before generating UI, AI must answer:

- What is the primary action?
- What is the user's goal?
- What is the friction?
- What is the success state?

If unclear, request clarification.
